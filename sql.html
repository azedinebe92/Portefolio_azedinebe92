Conception BD via interview

1. INTERVIEW

Voici l’interview de Yamina BENAZZI, employé du Crédit Agricole Assurance, en tant que Product Owner. Cette entreprise vend aux clients des contrats d’assurances dans différents domaines (assurance habitation, voiture…)

1) Avez-vous déjà réalisé la base de votre entreprise ?

Oui j’ai réalisé le modèle de données du Crédit Agricole Assurance. En effet l’entreprise a lancé un projet de refonte de leur SID (Système d’Information décisionnel) pour améliorer le pilotage de l’entreprise. L’ancienne Base de données développée avec Oracle devenait de moins en moins performante pour répondre aux différents demandes d’analyse des données des métiers.

2) A quoi servent vos bases de données dans votre entreprise ?

La base de données sert à :

  • Collecter les données en provenance des systèmes de gestion. Ces données peuvent provenir de différents systèmes de gestion (par ex gestion des contrats d’assurance, des personnes, des sinistres…).
  • Centraliser toutes les données qui viennent de différents systèmes dans une seule base de données pour les mettre à disposition des métiers pour leur permettre de faire des analyses par exemple :
    • suivre le chiffre d’affaire réalisé par agence et par mois
    • savoir si un client possède uniquement un contrat d’assurance pour son véhicule pour lui proposer d’autres contrats…

3) Combien de base de données disposez-vous ?

L’entreprise possède deux bases de données :

  • une base de données pour stocker les données des systèmes de gestion
  • une base de données décisionnelles où sont récupérées toutes les données des systèmes de gestion afin de pouvoir faire des analyses. Dans cette base de données, les données sont historisées.

4) Comment identifiez-vous un client de la base de données ?

La base de données fournit des informations organisées en table ou relation. Elle contient une table (CLIENT) qui permet de stocker l’ensemble des clients du Crédit Agricole. Cette table contient un identifiant unique par client (numClient) et des informations sur le client : nom, prénom, date de naissance, etc…

5) Avez-vous déjà rencontré des problèmes avec votre base de données ?

Les principaux problèmes rencontrés sont :

  • Problème de performance, les requêtes SQL permettant d’accéder aux données sont trop longues, ou tombent en erreur.
  • Problème de qualité de données :
    • Les tables contiennent des doublons (par ex un client est présent deux fois dans la même table avec deux noms différents)
    • Les relations entre les tables ne sont pas respectées, par ex on ne retrouve pas le client rattaché à un contrat
